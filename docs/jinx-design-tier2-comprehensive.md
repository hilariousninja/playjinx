# JINX — Comprehensive Design Reference (Tier 2)

Builds on Tier 1 with screen-by-screen breakdowns, component patterns, and copy conventions. ~25 pages.

---

## 1. Philosophy recap

See **Tier 1** for the full philosophy. In short:

1. Crowd prediction, not puzzle-solving
2. Social payoff over stats
3. Honesty over hype
4. Mobile-first, thumb-first

North star: *"Did you say the same thing as your mates?"*

---

## 2. Visual system

### Color tokens (HSL, defined in `index.css`)

- `--background` Cream `#F6F4EF`
- `--primary` Amber `#D97706` — jinxes, primary CTAs
- `--foreground` near-black for body text
- `--muted` warm grey for secondary text
- `--border` soft warm grey

Never use raw Tailwind color classes (`text-white`, `bg-black`). Always semantic tokens.

### Typography

- **Family:** Space Grotesk, all weights.
- **Header:** 52px display on every page top — brand consistency.
- **Body:** 16px base, 14px secondary.
- **Rule:** answers are NEVER truncated. Only recurring group names may truncate.

### Spacing & layout

- 4px base unit, generous padding on mobile (16–24px).
- Cards: rounded-xl, soft shadow, no harsh borders.
- Bottom nav is 3 tabs, persistent, 56px tall.

## 3. Screen-by-screen reference

### 3.1 Landing (`/`)

Three-section vertical flow on mobile: Brand → Hook → CTA → live example card. No marketing fluff above the fold beyond the hook.

### 3.2 Play (`/play`)

Single-screen play. One prompt card visible at a time. Input auto-focuses, auto-scrolls to next prompt on submit. After three answers → Results.

### 3.3 Results (`/results`)

CTA hierarchy:
1. **Brag Block** — "you jinxed N friends today" with shareable summary.
2. Friend/group comparisons.
3. Crowd stats (top % consensus).
4. Archive entry.

Colored border indicates rank vs crowd (top consensus = amber).

### 3.4 Archive (`/archive`)

Unified day feed. Today's card at top, past days below. Late play allowed (you can answer a past day).

### 3.5 Groups (`/groups`)

Hub of `ActiveGroupCard`s. Each card shows tinted activity bar (truthful — based on real submission count). Empty state encourages creating or joining.

### 3.6 Group Today (`/g/:slug`)

Three tabs at top:
- **Today** — today's prompts + jinx pairs
- **Members** — full vertical list (added because horizontal chips don't scale past 5)
- **History** — past days for this group

### 3.7 Group Pair (`/g/:slug/pair/:a/:b`)

Pairwise rivalry view: Twin / Sync / Wildcard / Opposite scoring across all played days.

### 3.8 Challenge Landing (`/c/:token`)

Friend-shared room. Plays like Play, but result compares directly against the sender.

### 3.9 Creator Dashboard (`/dashboard`)

Light theme. 5 tabs: Overview, Answers, Insights, Tuning, Data. Admin-only (`rajan.p@hotmail.co.uk`).

## 4. Component patterns

- **SlidePanel** — full-screen panels for secondary flows (z-index above page, below toasts).
- **vaul Drawer** — bottom-sheet drawer for quick actions; lower z-index than SlidePanel.
- **MyRoomCard** — surfaces challenge rooms the user created (so they can monitor activity).
- **PlayerIdentity** — persistent pill showing your display name (stored in localStorage).

---

## 5. Copy & voice

- **Tone:** dry, observational, never hype. "No jinxes yet today" not "Be the first!"
- **Pluralization:** "no jinxes" (not "no jinx"), "1 jinx", "2 jinxes".
- **Action verbs:** "Jinxed", "Said", "Played" — past tense, factual.
- **Never** use exclamation marks in default state. Reserve for genuine wins.

---

## 6. Core mechanics (summary)

- **One answer per day**, locked across all social contexts.
- **Three prompts per day**, generated by `pg_cron`, immutable once `total_players > 0`.
- **Normalization:** trim, lowercase, depluralize, fuzzy-merge near-duplicates.
- **Word freshness:** recency penalties prevent the same word reappearing too soon.

See **Tier 3** for the full mechanics spec, word lifecycle states, JINXability framework, and decision log.
